# 2026-03-10 Prompt 1.6

- timestamp: 2026-03-10
- prompt number: 1.6

## What Changed

- Extended `src/agentiad_recon/mmad.py` from one generic split/category indexer into a multi-source MMAD extractor indexer.
- Added source-specific split detection for DS-MVTec, MVTec-AD, MVTec-LOCO, VisA, and GoodsAD while preserving the existing legacy fixture layout.
- Added a small local smoke CLI and summary helper for source-by-split counts.
- Added `audit/check_prompt_1_6.py` to synthesize all five source layouts locally and validate canonical sample schema compliance.

## Why It Changed

- Prompt 1.6 requires the indexer to aggregate multiple MMAD benchmark sources under one extracted root for later SFT consumption.
- The existing Prompt 1.2 fixture path still had to remain intact because Prompt 1.3 through Prompt 1.5 already depend on the same sample contract.
- Heavy training was explicitly out of scope, so the work stayed in indexing plus lightweight acceptance only.

## Effects

- The repo can now recurse through a multi-source extracted MMAD root and normalize all supported datasets into the canonical sample schema.
- Canonical sample records still preserve source, split, category, anomaly labels, image references, masks, single-agent mode, and the fixed tool surface.
- Existing fixture-backed smoke tests continue to use the unchanged legacy layout.
