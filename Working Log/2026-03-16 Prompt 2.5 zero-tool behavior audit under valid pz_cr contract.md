Prompt Number

- Prompt 2.5

What Changed

- Added eval-side zero-tool behavior fields to prediction records so each sample now records its first protocol event, whether the first assistant output was terminal, ordered called tools, terminal-without-tool status, false/null zero-tool collapse status, and terminal answer turn index.
- Added `zero_tool_behavior_summary` to metrics, run summary, and run manifest artifacts.
- Added deterministic per-dataset and per-category zero-tool behavior sidecars.
- Added a read-only train-side audit helper in `src/agentiad_recon/behavior_audit.py` that reads exported SFT JSONL rows and measures pz_cr tool-first supervision strength without mutating data or retraining anything.

Why It Changed

- Prompt 2.4 smoke already proved the `pz_cr` prompt surface is valid.
- Yet `toolcall_rate` is still `0.0`, `PZ = 0.0`, and `CR = 0.0`.
- `parser_valid_rate` and `schema_valid_rate` are high, so the model is not crashing. It is behaviorally collapsing to direct terminal answers under a valid `pz_cr` contract.
- The missing evidence was:
  - how often turn-0 direct final answers happen
  - how often samples terminate with zero tools
  - how often that collapse is the false/null pattern
  - whether the pz_cr SFT data itself strongly teaches tool-first behavior

Effects

- The unified evaluator now emits decisive zero-tool behavior evidence without changing model outputs or tool contracts.
- Run artifacts can now distinguish tool-first behavior from direct-final-answer collapse per sample and in aggregate.
- The repo now includes a train-vs-eval comparison surface so the next milestone can target data supervision, inference policy, or both with evidence instead of guesswork.
