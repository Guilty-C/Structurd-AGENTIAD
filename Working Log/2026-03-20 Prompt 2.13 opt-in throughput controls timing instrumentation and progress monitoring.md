## What Changed

Prompt 2.13 keeps `src/agentiad_recon/baseline.py` as the single evaluation entrypoint and adds bounded opt-in throughput and observability controls inside the existing unified runtime:

- stage-specific generation budgets through `generation_stage_overrides`
- optional `emit_baseline_compare` and `emit_delta_report` tail toggles
- `artifact_level=throughput` sparse per-sample sidecar retention while preserving core predictions, metrics, summary, and manifest artifacts
- optional run-level timing instrumentation surfaced as `timing_summary`
- bounded progress monitoring with `progress_mode`, periodic stderr updates, and a machine-readable `progress_snapshot.json`

The patch also updates the maintained schemas, README, Prompt 2.13 regression tests, and the acceptance audit.

## Why It Changed

Full gate-on evaluation has become slow enough that exploratory runs need safe speed controls and better observability without forking the evaluator or weakening the paper-style reproduction target.

The goal of Prompt 2.13 is therefore operational rather than behavioral:

- measure where runtime wall time is going
- disable expensive compare/delta tail work when it is not needed
- reduce small-file sidecar volume for exploratory throughput runs
- allow tighter stage-specific decode budgets when explicitly requested
- make long runs monitorable in both interactive terminals and redirected logs

## What Effects The Changes Have

- Defaults remain conservative and preserve prior behavior when the new knobs are not set.
- Core artifacts are now written before optional compare/delta tail work, so summary/manifest availability no longer depends on that tail finishing first.
- Throughput mode can reduce raw-output and sidecar churn without changing metric semantics because summaries are computed from prediction records and counters, not by scanning sidecar directories.
- Timing summaries make generation, tool execution, parsing, file writing, and tail work visible at run level.
- Progress reporting now works in live terminal mode and in redirected-log mode, and it also writes a stable machine-readable snapshot file under the artifact root.
