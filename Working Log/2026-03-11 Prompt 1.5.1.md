# 2026-03-11 Prompt 1.5.1

- timestamp: 2026-03-11 13:10
- prompt number: 1.5.1

## What Changed

- Patched `src/agentiad_recon/sft.py` MS-Swift projection to render every `messages[*].content` as a string, never multimodal block arrays.
- Added `_render_swift_message_content(...)` to inject `<image>` placeholders in message order and keep tool-result payloads machine-readable JSON with explicit `image_placeholders` metadata.
- Updated `build_swift_records(...)` to construct top-level `images` as first-occurrence ordered paths, align placeholder traversal to that ordering, and fail fast on mismatch.
- Strengthened `src/agentiad_recon/ms_swift_adapter.py` with semantic validation after schema checks:
  - non-empty messages
  - string-only message content
  - non-empty image path strings
  - assistant-only `loss=true`
  - tool role `tool_name`/`call_id` consistency
  - `<image>` token count equals `len(images)`
- Updated `src/agentiad_recon/contracts/schemas/ms_swift_record.schema.json` so `messages[*].content` is `string`.
- Extended Prompt 1.5 checks in `tests/test_prompt_1_5_sft_export.py` and `audit/check_prompt_1_5.py` to validate string-only content, no list/dict content blocks, placeholder/image count equality, and first-occurrence image ordering.
- Added a minimal README note describing the Prompt 1.5.1 MS-Swift placeholder/image alignment contract.

## Why It Changed

- The Prompt 1.5 thin adapter emitted list-based multimodal blocks that conflict with current MS-Swift Qwen2.5-VL multimodal Agent expectations.
- The failure point moved to template encoding assertions, which indicates dataset shape incompatibility rather than framework/runtime faults.
- The fix needed to stay adapter-owned and minimal, preserving canonical trajectory semantics, message order, and loss masking while correcting only the projection/validation surface.

## Effects

- Exported MS-Swift records are now framework-ready for string-content multimodal encoding with `<image>` placeholders.
- Placeholder count is auditable and aligned with top-level `images` per record.
- Image ordering is stable and first-occurrence based across the dialogue.
- `pz_only` and `pz_cr` trajectory semantics and decisive-turn loss mask behavior remain intact, with stronger local acceptance gates for regression prevention.
