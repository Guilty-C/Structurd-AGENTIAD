Prompt Number

- Prompt 2.3

What Changed

- Added an explicit mixed-output normalization path in `src/agentiad_recon/tooling.py` so the runtime can deterministically choose the first legal `<tool_call>` when the same assistant turn also contains a final answer block.
- Wired that normalization decision into the existing `baseline.py` tool loop, writing one sidecar artifact per normalization event and threading the evidence into trace metadata, per-sample prediction metadata, `run_summary.json`, and `run_manifest.json`.
- Added Prompt 2.3 regression tests plus a lightweight acceptance script.

Why It Changed

- Prompt 2.0 and Prompt 2.1 already proved the real checkpoint-eval path and artifact chain were genuine, but remote full `pz_cr` evaluation still had a real blocker: `ToolContractError` was raised when one assistant output contained both `<tool_call>` and `<answer>`.
- That failure mode was also hard to audit because the raw output was not audit-visible enough once the run aborted. We needed a runtime-correct normalization path instead of letting the full eval crash.

Effects

- Mixed-output turns no longer abort the unified tool evaluator when a legal tool call is present.
- The first legal tool call is preserved and executed.
- The same-turn final answer is marked as premature for that turn and is explicitly recorded as discarded in structured evidence.
- Raw assistant output, normalization reason, selected tool call, and discarded-final-answer state are now preserved in run artifacts for later review.

Git-Diff-Style Summary

- `src/agentiad_recon/tooling.py`: adds `ProtocolDecision`, `normalize_protocol_turn(...)`, and structured audit payload generation while keeping `protocol_event(...)` strict for non-normalized callers.
- `src/agentiad_recon/baseline.py`: consumes the normalization helper inside the existing tool loop, writes sidecars next to raw outputs, and aggregates normalization counts into run-level artifacts.
- `tests/test_prompt_2_3_mixed_output_runtime_normalization.py`: adds parser/runtime and mocked tool-loop regression coverage.
- `audit/check_prompt_2_3.py`: checks that Prompt 2.3 hooks exist and that Prompt 2.x tests still pass.
- `README.md`: documents the mixed-output normalization behavior in the unified eval path.

Notes

- `ToolContractError` was the correct strict behavior for pure parsing, but it was the wrong runtime behavior for full eval once real mixed outputs appeared.
- This patch does not suppress or filter logs. It changes the runtime decision path and writes explicit evidence so the mixed-output case is reviewable after the run.
