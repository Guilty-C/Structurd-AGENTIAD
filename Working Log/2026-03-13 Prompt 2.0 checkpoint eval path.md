Why it changed
- The real blocker is no longer training. The blocker is wiring an already-trained SFT checkpoint into the existing baseline/eval path for real MMAD evaluation.
- The patch therefore stays in the thin adapter, CLI, schema, and audit layers and avoids changing trainer, export, or threshold-clean stages.
- Provenance fields were added so remote evaluation artifacts can show exactly which base model, adapter checkpoint, generation settings, and runtime flags produced the metrics.

Effects
- `baseline.py` can now accept a local base model path plus an optional LoRA adapter checkpoint and pass them to a real evaluation-capable backend

- Local validation remains lightweight through compile, unit tests, fixture smoke runs, dry-run surface checks, and mocked backend tests
