## What Changed

- Extended the Prompt 1.3 canonical inference runner into one tool-enabled path for `pz_only` and `pz_cr` instead of creating a second evaluator stack.

- Added Prompt 1.4 smoke tests, an acceptance checker, and README/config documentation updates.

## Why It Changed

- The master checklist requires tool-enabled inference reconstruction immediately after the non-tool baseline.
- The repository already had the sample waist, tool contracts, parser, trace storage, and baseline evaluator; this prompt had to extend that same path instead of duplicating it.
- Local validation still needed to remain mock-backed, deterministic, and auditable without pretending to be real model inference.

## Effects

- The repo now supports one canonical multi-turn tool loop that can run `PZ` only or `PZ` followed by `CR`, then produce the same strict final-answer record family used by the baseline path.
- Tool-aware runs now export per-sample prediction records, traces, tool-usage summaries, metrics, and structural delta-vs-baseline artifacts.
- Prompt 1.4 can be validated locally on fixtures with bounded scripted tool behavior and explicit malformed/disallowed tool handling.
